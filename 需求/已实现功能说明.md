# ScreenPulse 已实现功能说明

本文档面向项目负责人，用中文说明当前项目已经具备的产品功能、主要使用流程和实现边界。

## 一、项目当前定位

ScreenPulse 当前是一个网页端团队屏幕共享分析 MVP。

用户登录后可以创建或加入团队，并在选定团队中主动共享整个显示屏。浏览器会按团队设置的频率截取屏幕图片并上传到后端，后端保存截图、调用视觉模型生成截图观察结果，并按小时刷新成员工作总结。

当前系统保存的是截图、识别结果和小时级总结，不保存完整录屏，也不保存音频。

## 二、账号与登录

当前已实现：

- 用户注册。
- 用户登录。
- 登录状态恢复。
- 退出登录。
- 使用 JWT Bearer Token 调用后端接口。
- 登录成功后进入团队工作台。

注册时需要填写邮箱、姓名和密码。登录后，前端会把访问令牌保存在浏览器本地存储中，并在后续接口请求中自动携带。

## 三、团队功能

当前已实现：

- 查看自己已加入的团队。
- 创建团队。
- 创建团队后，创建者自动成为该团队的 `admin`。
- 选择当前要操作的团队。
- 使用邀请码加入团队。
- 加入团队后的默认角色是 `member`。

系统以团队为数据隔离单位。用户只能访问自己已经加入的团队。

## 四、团队角色与权限

当前团队内有两种角色：

| 角色 | 已实现权限 |
| --- | --- |
| admin | 管理团队邀请码、修改抽帧频率、查看团队成员、查看成员当前共享状态、查看成员小时级总结、管理成员角色、移除成员、查看和删除截图识别历史 |
| member | 在团队中共享自己的屏幕、查看自己的历史总结 |

已实现的权限限制：

- 非团队成员不能访问团队数据。
- `member` 不能生成邀请码。
- `member` 不能修改团队抽帧配置。
- `member` 不能查看其他成员总结。
- `member` 不能管理成员、邀请码状态或截图识别历史。
- `admin` 可以查看本团队成员列表和成员总结。
- `admin` 可以添加已注册用户为团队成员、修改成员角色、移除成员。
- `admin` 不能移除或降级团队内最后一个 active admin。

## 五、邀请码功能

当前已实现：

- `admin` 可以为团队生成邀请码。
- `admin` 可以查看团队邀请码列表。
- `admin` 可以启用或禁用邀请码。
- 用户可以输入邀请码加入团队。
- 邀请码有状态、使用次数和过期时间字段。
- 使用邀请码成功加入团队后，邀请码使用次数会增加。

当前前端会展示最近一次生成的邀请码和过期时间，方便管理员复制后发给成员。

独立管理员面板会展示邀请码列表、状态、使用次数和过期时间，并提供启用 / 禁用操作。

## 六、屏幕共享功能

当前已实现：

- 用户在选定团队后可以点击“开始共享”。
- 浏览器调用 `getDisplayMedia` 弹出系统屏幕共享授权。
- 前端要求用户共享整个显示屏，而不是单个窗口或标签页。
- 成功开始共享后，后端会创建一条团队内屏幕会话。
- 用户可以点击“停止共享”结束当前会话。
- 如果浏览器共享轨道被用户手动停止，前端也会尝试同步停止后端会话。
- 页面会显示当前共享状态、共享开始时间、最近上传时间、最近识别内容和最近活动描述。

当前共享链路是浏览器本地抽帧上传，不是 LiveKit Egress 录制链路。

## 七、截图抽帧与上传

当前已实现：

- 默认抽帧频率为 5 分钟一次。
- `admin` 可以修改团队抽帧频率。
- 抽帧频率支持秒、分钟、小时三个显示单位。
- 开始共享后会立即上传第一帧。
- 后续按团队当前设置定时上传截图。
- 后端只接受 PNG 和 JPEG 图片。
- 后端会限制上传图片大小和像素数量。
- 后端会把上传图片统一保存为 PNG 文件。

截图文件按团队和会话分目录保存，便于后续按团队、会话追踪。

## 八、截图识别与小时总结

当前已实现：

- 后端保存每张截图的捕获时间、尺寸、文件路径和所属团队/会话/用户。
- 后端调用兼容 OpenAI `/chat/completions` 风格的视觉模型接口分析截图。
- 视觉识别结果包含：
  - 可见软件和页面内容。
  - 用户主要活动描述。
- 每次上传截图后，后端会刷新该用户在当前小时内的小时级总结。
- 小时总结会记录：
  - 团队。
  - 用户。
  - 小时开始和结束时间。
  - 总结文本。
  - 参与总结的帧数。
  - 使用的模型名称。

如果没有配置模型接口，系统仍可运行，但截图识别和总结会退化为基础占位内容，不能代表真实视觉分析质量。

## 九、总结查看

当前已实现：

- `member` 可以查看自己的小时级历史总结。
- `admin` 可以查看团队成员列表。
- `admin` 可以选择某个成员查看该成员的小时级总结。
- 总结列表支持按日期筛选。
- 成员列表会展示成员最近总结。
- 成员列表会展示成员是否正在共享以及共享开始时间。

这部分功能用于让管理员了解团队成员是否正在共享，以及查看系统生成的工作摘要。

## 十、管理员专属面板

当前已实现独立管理员界面：

- 页面入口：`/admin`。
- 本地可通过 `npm run dev:admin` 在 `3011` 端口单独启动，便于和普通用户界面区分。
- 只展示当前登录用户拥有 `admin` 权限的团队。
- 可查看所有团队成员的实时共享状态、共享开始时间和最近总结。
- 可查看全队或指定成员的小时级总结。
- 可修改团队抽帧频率。
- 可生成、查看、启用和禁用邀请码。
- 可查看历史截图元数据、识别内容和活动描述。
- 可删除历史截图及其识别结果，删除后后端会刷新对应小时总结。
- 可添加已注册用户、移除成员、修改成员 `admin` / `member` 角色。

## 十一、审计日志

当前后端已经有团队级审计日志表和审计服务。

系统会为关键团队操作记录审计事件，例如：

- 创建团队。
- 生成邀请码。
- 加入团队。
- 修改团队设置。
- 修改成员角色。
- 移除成员。
- 修改邀请码状态。
- 删除截图记录。
- 开始屏幕会话。
- 停止屏幕会话。

目前审计日志主要用于后端追踪和后续扩展，前端还没有单独的审计日志查看页面。

## 十二、LiveKit 相关能力

当前已实现：

- 后端保留 LiveKit token 签发接口。
- 如果配置了 LiveKit 相关环境变量，可以签发团队房间 token。

当前未作为主流程使用：

- LiveKit 推流。
- LiveKit Egress 录制。
- 基于 LiveKit 的服务端抽帧。

当前 MVP 主流程仍然是浏览器本地截图上传。

- 管理员专属面板页。

团队工作台中已经集成：

## 十四、数据保存范围

当前会保存：

- 用户信息。
- 团队信息。
- 团队成员关系。
- 邀请码。
- 团队抽帧设置。
- 屏幕共享会话。
- 截图文件。
- 截图识别结果。
- 小时级总结。
- 审计日志。

当前不会保存：

- 完整录屏。
- 音频。
- 浏览器实时媒体流。

## 十五、当前实现边界

当前项目仍是 MVP，已经具备核心闭环，但还有一些边界需要明确：

- 截图分析调用是同步执行的，上传请求会等待模型调用完成。
- 后端使用 SQLite 和同步 SQLAlchemy session。
- 没有独立任务队列。
- 没有单独的审计日志前端页面。
- 没有团队删除功能。
- LiveKit 只保留 token 能力，不是当前主链路。

## 十六、核心使用流程

### 1. 管理员创建团队并邀请成员

```text
注册或登录
→ 创建团队
→ 自动成为 admin
→ 生成邀请码
→ 把邀请码发给成员
```

### 2. 成员加入团队并开始共享

```text
注册或登录
→ 输入邀请码加入团队
→ 选择团队
→ 点击开始共享
→ 浏览器授权共享整个显示屏
→ 系统立即上传第一帧
→ 后续按团队频率持续抽帧上传
```

### 3. 管理员查看成员状态和总结

```text
登录
→ 打开管理员面板
→ 选择拥有 admin 权限的团队
→ 查看成员实时共享状态
→ 查看成员是否正在共享
→ 选择成员
→ 查看小时级总结
```

### 4. 管理员管理团队

```text
登录
→ 打开管理员面板
→ 选择团队
→ 管理抽帧设置 / 邀请码 / 成员角色 / 截图识别历史
```

### 5. 成员查看自己的历史总结

```text
登录
→ 选择团队
→ 查看我的历史总结
→ 按日期筛选小时级总结
```
